Highland.js

API Documentation

Fixing JavaScript with JavaScript.

The Highland project is an experiment in replacing some of the most dangerous and unpredictable features of JavaScript using one of the language's best assets: Functions. Highland values predictability and expressiveness over speed, knowing that you can break into regular JavaScript when speed is essential, and elsewhere limit your exposure to it's sometimes unpredictable results.

This is not a new language that compiles to JavaScript, it is a collection of many small functions to help you write expressive, more functional code with fewer side-effects.

Functions

curry f args... -> Function(...)

Transforms a function with specific arity (all arguments must be defined) in a way that it can be called as a chain of functions until the arguments list is saturated.

This function is not itself curryable.

Parameters

  • f Function - the function to curry
  • args.. - any number of arguments to pre-apply to the function

Example

fn = curry(function (a, b, c) {
    return a + b + c;
});

fn(1)(2)(3) == fn(1, 2, 3)
fn(1, 2)(3) == fn(1, 2, 3)
fn(1)(2, 3) == fn(1, 2, 3)

ncurry n fn args... -> Function(...)

Same as curry but with a specific number of arguments. This can be useful when functions do not explicitly define all its parameters.

This function is not itself curryable.

Parameters

  • n Number - the number of arguments to wait for before apply fn
  • fn Function - the function to curry
  • args... - any number of arguments to pre-apply to the function

Example

fn = ncurry(3, function () {
    return Array.prototype.join.call(arguments, '.');
});

fn(1, 2, 3) == '1.2.3';
fn(1, 2)(3) == '1.2.3';
fn(1)(2)(3) == '1.2.3';

compose a -> b -> Function(x)

Creates a composite function, which is the application of function 'a' to the results of function 'b'.

Parameters

  • a Function - the function to apply to the result of b(x)
  • b Function - the function to apply to x

Example

var add1 = add(1);
var mul3 = mul(3);

var add1mul3 = compose(mul3, add1);
add1mul3(2) == 9

apply f -> args -> result

Applies function f with arguments array args. Same as doing fn.apply(this, args)

Parameters

  • f Function - the function to apply the arguments to
  • args Array - an array of arguments to apply

Example

apply(add, [1,2]) == 3
apply(mul)([3,3]) == 9

flip f -> x -> y -> result

Evaluates the function f with the argument positions swapped. Only works with functions that accept two arguments.

Parameters

  • f Function - function to flip argument application for
  • x - parameter to apply to the right hand side of f
  • y - parameter to apply to the left hand side of f

Example

div(2, 4) == 0.5
flip(div)(2, 4) == 2

Operators

eq a -> b -> Boolean

Tests for equality using ===

Parameters

  • a Number
  • b Number

Example

eq(1,1) == true
eq(1,2) == false

eqv a -> b -> Boolean

Tests if the values of a and b are equivalent. With objects and arrays this function will recursively test sub-properties in order to determine equivalence. The eq function when applied to two instances of the same prototype will return false, this function will return true.

Parameters

  • a - any value
  • b - any value

Example

eqv({a: 1}, {a: 1}) == true
eqv({a: 1, b: {c: 2}}, {a: 1, b: {c: 3}}) == false

ne a -> b -> Boolean

Tests for inequality using !==

Parameters

  • a - any value
  • b - any value

Example

ne(1,1) == false
ne(1,2) == true

not a -> Boolean

Tests if a is not truthy using !, this only works with Boolean values.

Parameters

  • a Boolean - the boolean value to return the inverse of

Example

not(true) == false
not(false) == true

lt a -> b -> Boolean

Tests if a is less than b. This is not a simple wrapper for the '<' operator, and will only work with Numbers, Strings and Arrays (containing any of these three types). Both a and b must be of the same data type, you cannot compare a Number with a String, for example. However, you can compare two arrays which both have a Number as the first argument and a String as the second, and so on.

Parameters

  • a Number|String|Array
  • b Number|String|Array

Example

lt(2,4) == true
lt(5,1) == false
lt(3,3) == false

gt a -> b -> Boolean

Tests if a is greater than b. This is not a simple wrapper for the '>' operator, and will only work with Numbers, Strings and Arrays (containing any of these three types). Both a and b must be of the same data type, you cannot compare a Number with a String, for example. However, you can compare two arrays which both have a Number as the first argument and a String as the second, and so on.

Parameters

  • a Number|String|Array
  • b Number|String|Array

Example

gt(2,4) == false
gt(5,1) == true
gt(3,3) == false

le a -> b -> Boolean

Tests if a is less than or equivalent to b.

Parameters

  • a Number|String|Array
  • b Number|String|Array

Example

le(2,4) == true
le(5,1) == false
le(3,3) == true

ge a -> b -> Boolean

Tests if a is greater than or equivalent to b.

Parameters

  • a Number|String|Array
  • b Number|String|Array

Example

gt(2,4) == false
gt(5,1) == true
gt(3,3) == true

and a -> b -> Boolean

Tests if both a and b are true using &&. However, unlike the && operator, this will only work with Boolean arguments. It has no concept of 'truthy' and 'falsey'.

Parameters

  • a Boolean
  • b Boolean

Example

and(true, true) == true
and(false, true) == false
and(false, false) == false

or a -> b -> Boolean

Tests if either a or b are true using ||. However, unlike the || operator, this will only work with Boolean arguments. It has no concept of 'truthy' and 'falsey'.

Parameters

  • a Boolean
  • b Boolean

Example

or(true, true) == true
or(false, true) == true
or(false, false) == false

add a -> b -> Number

Adds a and b using +. This only works with Numbers, it does not also perform string concatenation. For that, use the concat function.

Parameters

  • a Number
  • b Number

Example

add(1,2) == 3
add(5,5) == 10

sub a -> b -> Number

Subtracts b from a using -. This only works with Numbers.

Parameters

  • a Number
  • b Number

Example

sub(2,1) == 1
sub(5,5) == 0

mul a -> b -> Number

Multiplies a and b using *. This only works with Numbers.

Parameters

  • a Number
  • b Number

Example

mul(2,1) == 2
mul(5,5) == 25

div a -> b -> Number

Divides a by b using /. This only works with Numbers.

Parameters

  • a Number
  • b Number

Example

div(4,2) == 2
div(15,5) == 3

rem a -> b -> Number

Returns the amount left over after dividing integer a by integer b. This is the same as the % operator, which is in fact the remainder not modulus. However, this function will only work with Number arguments.

Parameters

  • a Number
  • b Number

Example

rem(-1, 5) == -1

mod a -> b -> Number

The modulus of a and b, this is NOT the same as the % operator in JavaScript, which actually returns the remainder. See the rem function if you want compatible behaviour with %.

Parameters

  • a Number
  • b Number

Example

mod(-1, 5) == 4

Tests if obj is an Object. This differs from other isObject implementations in that it does NOT return true for Arrays, Functions or Strings created using the String() constructor function.

Parameters

  • x - the value to test

isObject(function(){}) == false

max x -> y -> Number

Returns the highest of two values. Works with Numbers, Strings or Arrays.

Parameters

  • x Number|String|Array
  • y Number|String|Array

Example

max(1,5) == 5
max([1,2,3],[2,3,4]) == [2,3,4]

min x -> y -> Number

Returns the lowest of two values. Works with Numbers, Strings or Arrays.

Parameters

  • x Number|String|Array
  • y Number|String|Array

Example

min(1,5) == 1
min([1,2,3],[2,3,4]) == [1,2,3]

compare x -> y -> -1 | 0 | 1

Compares two values, returning -1 if x it less than y, 0 if the values are equivalent, and 1 if x is greater than y. Works with Numbers, Strings or Arrays.

Parameters

  • x Number|String|Array
  • y Number|String|Array

Example

compare(1,2) == -1
compare([1,2,3],[1,2,3]) == 0
compare(5,3) == 1

Lists

cons x -> xs -> Array | String

Creates a new Array by prepending an element to an existing array. Can also be used with Strings.

Parameters

  • x - the value to prepend to the xs in the new array
  • xs Array|String - the tail of the new array

Example

cons(0, [1,2,3]) == [0,1,2,3]

append x -> xs -> Array | String

Creates a new Array by appending an element to an existing array. Can also be used to append a character to a String.

Parameters

  • x - the value to append to the xs in the new array
  • xs Array|String - the init of the new array

Example

append(4, [1,2,3]) == [1,2,3,4]

head xs -> x

Returns the first element of a non-empty Array. Can also be used with Strings.

Parameters

  • xs Array|String - the array to return the first element from

Example

head([1,2,3,4]) == 1

last xs -> x

Returns the last element of a non-empty Array. Can also be used with Strings.

Parameters

  • xs Array|String - the array to return the last element from

Example

last([1,2,3,4]) == 4

tail xs -> Array

Returns a new Array without the first element of the original non-empty Array. Can also be used with Strings.

Parameters

  • xs Array|String - the array to return the tail of

Example

tail([1,2,3,4]) == [2,3,4]

init xs -> Array

Returns a new Array without the last element of the original non-empty Array. Can also be used with Strings.

Parameters

  • xs Array|String - the array to return the init of

Example

init([1,2,3,4]) == [1,2,3]

empty xs -> Boolean

Returns true if the Array or String is empty.

Parameters

  • xs Array|String - the array to test

Example

empty([]) == true
empty([1,2,3]) == false

length xs -> Number

Returns the length of an Array or String.

Parameters

  • xs Array|String - the array to return the length of

Example

concat a -> b -> Array | String

Adds the elements of one Array to another, returning a new Array. Also works on strings.

Parameters

  • a Array|String
  • b Array|String

Example

concat([1,2], [3,4]) == [1,2,3,4]
concat('abc', 'def') == 'abcdef'

foldl f -> z -> xs -> result

Takes a combining function f an Array xs and an initial value z, and boils down a list of values into a single value. The z arguments is the initial state of the reduction, and each successive step of it should be returned by iterator.

Parameters

  • f Function - the combining function
  • z - the inital value
  • xs Array|String - the array to combine

Example

foldl(add, 1, [2,3,4]) == 10

foldl1 f -> xs -> result

Same as foldl but uses the first element as the initial value instead of the z argument.

Parameters

  • f Function - the combining function
  • xs Array|String - the array to combine

Example

foldl1(add, [1,2,3,4]) == 10

foldr f -> z -> xs -> result

A fold starting from the right side, or end of, the Array or String.

Parameters

  • f Function - the combining function
  • z - the initial value
  • xs Array|String - the array to combine

Example

foldr(add, 4, [1,2,3]) == 10

foldr1 f -> xs -> result

Same as foldr but uses the first element as the initial value instead of the z argument.

Parameters

  • f Function - the combining function
  • xs Array|String - the array to combine

Example

foldr1(add, [1,2,3,4]) == 10
